# 테스트 프로젝트2

## 웹 UI 레이아웃 개요
- **중앙 게임판**: 정사각형 비율 캔버스. 데스크톱에서는 가로 폭의 60~70%를 차지하며, 모바일에서는 화면 전체 폭을 사용.
- **상단 상태 표시줄(HUD)**: 좌측에 현재 라운드, 우측에 남은 시간/점수. 진척도를 보여주는 얇은 프로그레스 바 포함.
- **우측 사이드 패널** (데스크톱 기준): 실시간 로그, 다음 행동 가이드, 보상 조건 등을 카드 형태로 스크롤 가능하게 배치.
- **하단 컨트롤 영역**: 핵심 액션 버튼 2~3개와 보조 버튼을 그룹화. 모바일에서는 게임판 아래로 접히는 아코디언 패널 형태.

## 와이어프레임 스케치
```
┌───────────────────────────────────────────────┐
│ LOGO / 게임 제목                              │
├───────────────┬───────────────────────────────┤
│ 상태 정보     │ 라운드 · 점수 · 남은 시간     │
│ 프로그레스바  │■■■■■■□□□□□□□□□□□□□□          │
├───────────────┴───────────────────────────────┤
│                                               │
│                 게임판 캔버스                 │
│                                               │
├───────────────────────┬───────────────────────┤
│ 메인 액션 버튼 영역  │ 사이드 패널           │
│ [시작] [일시정지]    │ - 이벤트 로그         │
│ [재시도]             │ - 다음 목표           │
│ 보조 버튼 토글       │ - 힌트/도움말 카드    │
└───────────────────────┴───────────────────────┘
```

## 반응형 동작 가이드
- 960px 이상: 좌우 2열 구성(게임판 + 사이드 패널). 컨트롤 영역을 게임판 아래 전체 폭으로 배치.
- 600~959px: 사이드 패널을 게임판 아래 카드 스택으로 이동, 컨트롤 영역은 고정 하단 바로 축소.
- 599px 이하: 순서 `상단 HUD → 게임판 → 컨트롤 버튼 → 접을 수 있는 세부 정보`. 주요 버튼은 전체 폭, 보조 기능은 `더 보기`로 묶기.

## 상호작용 요소 정리
- **상태 표시줄**: 실시간 업데이트, 경고 상황(시간 부족 등) 시 배경색 변환.
- **주 버튼 그룹**: 1차 액션(예: `턴 종료`)을 강조 색으로, 나머지는 중립 톤. 버튼 간 최소 16px 간격 유지.
- **사이드 패널 카드**: 섹션별 아이콘과 짧은 제목, 최대 2~3줄 요약. 중요 HUD 알림은 토스트로도 노출.
